## 3. 現在の課題と改善ポイント

### 3-1. 🔴 globals.css の巨大化（35KB / 2,009行）

> [!CAUTION]
> v7時点の1,266行から2,009行へ**59%増加**しています。UI改善コミットでスタイルが継続的に積み増しされた結果、単一CSSファイルとしては保守限界に近づいています。

**現在の構成（推定）:**
- CSS変数・リセット: ~60行
- ログイン系: ~120行
- サイドバー・ナビゲーション: ~300行
- レイアウト（admin-shell, split, grid）: ~200行
- パネル・テーブル・ボタン・フォーム: ~400行
- メトリクス・ワークスペース: ~150行
- ポータル系: ~200行
- 給与・帳票系: ~200行
- アニメーション: ~80行
- レスポンシブ（メディアクエリ）: ~300行

**推奨分割案:**

```
app/
  globals.css          — CSS変数、リセット、共通要素（~200行）
  layout.css           — admin-shell, split, grid, sidebar（~400行）
  components.css       — パネル、テーブル、ボタン、フォーム、メトリクス（~500行）
  portal.css           — 職員ポータル専用スタイル（~300行）
  payroll.css          — 給与・帳票専用スタイル（~200行）
  responsive.css       — 全メディアクエリ集約（~300行）
  animations.css       — キーフレーム・トランジション（~100行）
```

### フェーズ A: globals.css の分割（最優先 / 1-2日）

> [!IMPORTANT]
> 2,009行の単一CSSファイルは、UI改善のたびにコンフリクトや意図しないスタイル上書きのリスクが高まります。機能追加より先に分割することを推奨します。

**手順:**

1. `app/globals.css` → CSS変数・リセット・共通要素のみに縮小

4. `app/portal.css` ← portal-home, employee-portal専用スタイル

7. `app/animations.css` ← `@keyframes` とトランジション定義
8. `app/layout.tsx` で全CSSファイルを import
9. `npm run build` で確認

**完了条件:** 各CSSファイルが500行以下、`npm run build` 成功
