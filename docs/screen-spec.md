# 職員打刻・休暇申請・給与明細配信アプリ 画面項目定義書

## 1. 目的

本書は、Windows打刻アプリ、職員向けモバイルアプリ、管理者向けWebアプリの主要画面における表示項目、入力項目、操作、バリデーションを整理する。

## 2. 共通方針

- 画面IDは `A` = 打刻アプリ、`M` = モバイル、`W` = 管理Web とする
- 必須項目は UI 上で明示する
- エラーメッセージは利用者が理解できる文言にする
- 操作成功時は結果が明確に分かるフィードバックを表示する

## 4.3 M-03 休暇申請入力

### 入力項目

| 項目名 | 種別 | 必須 | バリデーション | 説明 |
| --- | --- | --- | --- | --- |
| 休暇区分 | 選択 | YES | PAID, ABSENCE, SPECIAL | 申請区分 |
| 開始日 | 日付 | YES | 当日以前も可否は運用で決定 | 開始日 |
| 終了日 | 日付 | YES | 開始日以降 | 終了日 |
| 申請単位 | 選択 | YES | FULL, HALF | 1日または半日 |
| 半日区分 | 選択 | 条件付き | AM, PM | 半日の場合必須 |
| 理由 | 複数行入力 | NO | 500文字以内 | 備考 |

### 表示項目

| 項目名 | 種別 | 必須 | 説明 |
| --- | --- | --- | --- |
| 有給残数 | 表示 | 条件付き | 区分が有給の場合のみ表示 |
| 申請日数 | 表示 | - | 自動計算結果 |

### バリデーション

- `開始日 <= 終了日`
- 半日申請時は開始日と終了日を同日に制限
- 欠勤は初期MVPでは半日申請不可
- 有給は残数不足時に申請不可

### 操作

- 確認
- 申請
- キャンセル

## 6. 主な画面制御ルール

### 6.1 休暇申請

- 有給選択時のみ有給残数を表示する
- 欠勤選択時は半日項目を非表示または入力不可にする
- 半日選択時は終了日を開始日と同値に固定する

### 6.2 給与明細

- 未公開の明細は職員アプリ側一覧に表示しない
- 閲覧時に履歴を自動登録する

### 6.3 打刻

- オフライン時は結果画面に「一時保存済み」を明示する
- 未登録カード時は職員名を表示しない
- 二重打刻警告時も、運用方針に応じて受付可否をAPI応答で制御する
