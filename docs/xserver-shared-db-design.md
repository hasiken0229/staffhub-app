# Xserver共有レンタルサーバー向け DB設計書

## 1. 目的

本書は、`Xserver共有レンタルサーバー` 上で運用することを前提に、`MySQL` 向けのDB設計を定義する。

対象業務は以下とする。

- カード打刻
- 休暇申請と承認
- 給与明細PDF配信
- 通知
- 監査ログ

## 2. 前提

- DBMS: `MySQL 8系`
- 文字コード: `utf8mb4`
- 照合順序: `utf8mb4_unicode_ci`
- 日時型: `datetime`
- 主キー: `bigint unsigned`
- アプリフレームワーク: `Laravel`

## 3. 設計方針

- イベント系は更新より `追加中心`
- カード紐付けや給与明細は `履歴を残す`
- 有効カードは1枚のみ有効という業務ルールは `アプリ側制御` を基本とする
- 日次勤怠は `attendance_events` から集計し、`attendance_daily` に反映する
- PDF本体はDBに保存せず、保存パスのみ保持する

## 4. PostgreSQL版からの主な変更点

- `timestamp with time zone` は `datetime` に変更
- `jsonb` は `json` に変更
- `partial index` は使わず、アプリ側で排他制御
- `check constraint` 依存の業務制約はアプリとバリデーションに寄せる
- `generated by default as identity` は `AUTO_INCREMENT` に変更

## 7. インデックス方針

- 打刻検索は `employee_id + occurred_at`
- 当日一覧は `target_date`
- 通知は `employee_id + is_read + sent_at`
- 管理画面の一覧系は職員番号、状態、発生日の検索を重視

## 8. 共有サーバー運用上の注意

- `employee_cards` の有効カード一意性はMySQLだけで完全保証しづらいため、カード登録処理をトランザクション化する
- PDF保存先は `public_html` 外に置く
- バックアップは DB ダンプ + PDFファイル の両方を取る
