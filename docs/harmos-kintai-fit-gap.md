# HARMOS勤怠 機能棚卸しと改善計画

作成日: 2026-03-23

## 1. 前提

- 現在の運用元は `HARMOS勤怠`
- 新システムの利用先は `当園のみ`
- `フレックスタイム` は不要
- `管理者も職員の一人` という運用を前提にする
- 優先するのは `打刻`, `日次確認`, `休暇申請`, `承認`, `給与明細/賞与明細`, `CSV運用`
- 特に `有給休暇運用` は HARMOS そのままではなく、当園向けに改善する

## 2. HARMOS勤怠で公式確認できた主な機能

以下は 2026-03-23 時点で、公式FAQ・公式サービスページで確認できた内容です。

### 2-1. 打刻

- `ICカード打刻アプリ` があり、交通系ICカードや社員証などをかざして本人識別し、打刻できる
- `勤怠アプリ` があり、個人向けに `打刻`, `日次勤怠`, `承認`, `申請` などを利用できる
- 勤怠アプリでは `申請・承認漏れ` や `36協定アラート` をプッシュ通知で受け取れる

### 2-2. 休暇・届出

- `有給`, `代休`, `特別休暇` などの残数調整ができる
- `1時間単位有給` は有料プランのオプション機能
- `過去に遡った有給残数の自動付与` はできない
- 休暇管理を別運用にする場合は、システム設定で `年次有給休暇等・管理` を外して無効化できる
- 届出機能利用時は、設定によって日次勤怠編集で勤務区分を選べなくなることがあり、制御設定が必要

### 2-3. 権限

- `一般利用者`
- `承認者`
- `利用者・承認者`
- `勤怠システム担当`

### 2-4. 給与明細

- `ハーモス給与明細` は Web給与明細サービスとして提供されている
- `給与データはCSVでインポート` できる
- `支給日` と `公開日` の指定ができる
- 職員は `スマホに最適化された画面` で明細確認ができる

## 3. HARMOSから当園向けに引き継ぐべき機能

### 3-1. そのまま引き継ぐ

- `カード打刻`
- `職員本人の打刻履歴確認`
- `管理者の日次勤怠確認`
- `休暇申請`
- `1段階承認`
- `給与明細/賞与明細のWeb配信`
- `職員CSV取込`
- `給与CSV/賞与CSV取込`
- `権限分離`
- `監査ログ`

### 3-2. 当園向けに簡素化する

- `フレックスタイム` は外す
- `複雑な勤務体系` は外す
- `多段承認` は外す
- `複数拠点向け設定` は最小化する
- `汎用的すぎる届出体系` は絞る

### 3-3. 当園向けに強化する

- `有給休暇管理`
- `休暇の履歴管理`
- `CSV取込の分かりやすさ`
- `管理画面の運用性`
- `職員ログインと配信導線`

## 4. 当園向けの改善方針

### 4-1. 打刻

HARMOSは多様な打刻手段を持っていますが、当園では手段を絞った方が運用が安定します。

#### 採用方針

- 打刻は `Windows常駐アプリ + RC-S380 + ICカード` に統一
- 打刻種別は `出勤 / 退勤` を基本とする
- ネット断時はローカル保持して再送
- 管理者は `当日打刻` と `日次勤怠` を同一画面で確認できるようにする

#### 将来候補

- 打刻漏れ通知
- 異常打刻アラート
- 勤怠修正申請

### 4-2. 休暇

ここが HARMOS より改善すべき最優先領域です。

#### HARMOSで見えた制約

- 過去遡及の自動付与ができない
- 残数調整に寄る運用になりやすい
- 履歴ベースで追いにくい

#### 当園向けの改善仕様

- 残数は `現在値保持` ではなく `付与履歴 + 消化履歴 + 調整履歴` から計算
- `過去日付` の付与・調整登録を可能にする
- `承認済み` のみ残数へ反映
- `取消 / 差戻し / 却下` 時は自動で巻き戻す
- 半日単位を標準対応
- `1時間単位有給` は今すぐ必須ではないが、設計上は追加可能にしておく
- `特休` はテンプレート化して理由分類しやすくする
- `欠勤` は残数管理対象外として申請・記録のみ行う

#### 必ず持つべき履歴

- 付与
- 消化
- 手動調整
- 承認
- 差戻し
- 却下
- 取消

### 4-3. 権限

HARMOSの4権限は考え方として参考になりますが、当園向けには次の3ロールで十分です。

#### 推奨ロール

- `職員`
  - 自分の打刻確認
  - 休暇申請
  - 明細確認
- `承認者`
  - 休暇承認
  - 勤怠確認
- `システム管理者`
  - 職員管理
  - カード管理
  - CSV取込
  - 明細生成/公開
  - 監査ログ確認

#### 補足

- 管理者も職員として利用するため、`1ユーザー1役割` ではなく `職員属性 + 権限属性` の組み合わせで設計する
- つまり `管理者は職員画面も確認できる` 設計に寄せる

### 4-4. 給与明細

HARMOS給与明細は `CSVインポートでWeb配信` という方向です。  
当園向けには、そこを一歩進めて `CSV -> PDF自動生成 -> 配信` まで一気通貫にします。

#### 採用方針

- 給与CSV・賞与CSVを管理画面から取込
- `対象年月`, `公開日時`, `種別` を管理画面で指定
- 職員番号で自動紐付け
- 職員ごとのPDFを自動生成
- 職員はスマホ/ブラウザで一覧・詳細・ダウンロード

#### さらに欲しい改善

- エラー行を画面で一覧化
- 雛形CSVのダウンロード
- 既存明細との重複チェック
- 公開前プレビュー

## 5. 現在の新システムで既にできていること

### 実装済み

- `RC-S380 + Windows打刻`
- `カード登録`
- `認証`
- `職員/管理者の権限制御`
- `休暇申請`
- `1段階承認`
- `給与PDF手動登録`
- `給与CSV -> PDF自動生成`
- `賞与CSV -> PDF自動生成`
- `職員CSV取込`
- `管理画面からCSV取込`

### 未完成または改善余地あり

- 有給付与・調整履歴の運用画面
- 職員ログイン一括発行
- 打刻漏れ/異常打刻アラート
- レポート出力
- 管理UIの最終調整

## 6. 改善計画案

### フェーズ1. HARMOS互換の運用を固める

#### 目的

- 現行運用から違和感なく切り替えられるようにする

#### 実施内容

- 打刻ルールの確定
- 権限の最終確定
- 休暇区分の最終確定
- 職員CSVの確定版整備
- 給与CSV/賞与CSVの正式フォーマット固定

#### 完了条件

- 毎月の `打刻 / 休暇 / 給与明細` を現行と同じ粒度で回せる

### フェーズ2. 有給休暇の改善

#### 目的

- HARMOSより扱いやすい有給運用へ置き換える

#### 実施内容

- `leave_grants` の整備
- `leave_adjustments` の整備
- 付与・消化・調整の時系列表示
- 残数計算ロジックの履歴ベース化
- 取消・差戻し時の巻き戻し
- 特休テンプレート管理

#### 完了条件

- 過去日付を含む有給残数の説明が履歴だけでできる

### フェーズ3. 管理運用の省力化

#### 目的

- 管理者の月次作業を短くする

#### 実施内容

- 職員ログイン一括発行
- 初期パスワード再発行
- CSV取込結果の画面表示強化
- 明細公開前チェック
- 公開後の未閲覧確認

#### 完了条件

- 月次の明細公開作業が画面上だけで完結する

### フェーズ4. 勤怠チェック強化

#### 目的

- 打刻漏れや確認漏れを減らす

#### 実施内容

- 打刻漏れ検知
- 連続打刻異常検知
- 休暇承認滞留の一覧化
- 当日未退勤の見える化

#### 完了条件

- 管理者が朝夕の確認で異常を拾える

### フェーズ5. 移行完了

#### 目的

- HARMOSを実務上止めても困らない状態にする

#### 実施内容

- 運用手順書作成
- バックアップ手順確立
- 初期データ整備
- 本番切替リハーサル

#### 完了条件

- 当園の通常月次運用を新システムだけで回せる

## 7. 次に着手すべき順番

1. `有給付与・調整履歴` の詳細設計
2. `職員ログイン一括発行` の仕様化
3. `賞与CSVの全件取込確認`
4. `CSV取込結果画面` の強化
5. `打刻漏れ/異常打刻アラート`

## 8. 今回の結論

- HARMOSから引き継ぐべき中心機能は `打刻`, `申請承認`, `給与明細配信`, `権限管理`
- 当園では `フレックス` や `多様な打刻手段` より、`単純で止まりにくい運用` を優先すべき
- 一番改善価値が高いのは `有給休暇管理`
- 新システムは既に HARMOS の主要業務をかなり代替できる段階にあり、次は `有給履歴設計` が最重要
